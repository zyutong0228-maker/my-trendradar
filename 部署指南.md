# TrendRadar 热点新闻聚合助手 · 部署指南

> 这份指南是给**非程序员**看的。跟着一步步点，不用懂代码。
> 部署完成后，每天早 8 点、晚 8 点，你的企业微信群就会自动收到一份小升初相关的热点新闻简报。

---

## 📋 你将完成什么

- ✅ 把这个项目放到你自己的 GitHub 账号下
- ✅ 配一个企业微信群机器人，拿到"推送地址"
- ✅ 在 GitHub 上设好"密码保险箱"（Secrets）
- ✅ 开启自动运行开关（GitHub Actions）
- ✅ 顺便开启在线网页版（GitHub Pages）

**总耗时**：约 30 分钟。全程只在浏览器里点点点。

---

## 🧰 准备工作

1. **一个 GitHub 账号**（没有就去 https://github.com 免费注册）
2. **一个企业微信群**，你是群管理员
3. **本地电脑**：什么操作系统都行

---

## 步骤 1️⃣：把项目放到你自己的 GitHub

### 方式 A（推荐）：用"一键导入"

1. 打开浏览器，登录 GitHub
2. 访问 https://github.com/new/import
3. **"Your old repository's clone URL"** 里填：
   ```
   https://github.com/sansan0/TrendRadar
   ```
4. **"Repository Name"** 起个名字，比如 `my-trendradar`
5. 选择 **Private**（私有）——⚠️ 强烈建议私有，防止 webhook 泄露
6. 点 **Begin import**，等 1~2 分钟导入完成

### 方式 B：本地已改好后推上去

如果你本地这个 `热点新闻聚合助手` 文件夹已经让 Claude 改过（关键词、配置），想把本地版本推到 GitHub：

1. 先在 GitHub 上创建一个空的私有仓库：https://github.com/new
   - 名字任意，例如 `my-trendradar`
   - Private
   - **不要勾** "Initialize with README"（因为本地已有代码）
2. 回到本地终端，进入项目目录：
   ```bash
   cd "/Users/mac/Claude code_test/热点新闻聚合助手"
   git remote remove origin
   git remote add origin https://github.com/你的用户名/my-trendradar.git
   git branch -M main
   git push -u origin main
   ```
3. 刷新 GitHub 页面，应该能看到代码了

## 步骤 2️⃣：拿企业微信群机器人的 Webhook 地址

1. 打开企业微信电脑版
2. 进入你要接收推送的群 → 右上角齿轮（群设置）
3. 下拉找到 **"群机器人"** → **添加**
4. 起个名字，比如 "小升初热点"，头像可选可不选
5. 创建完成后，点机器人，页面会显示一串 **Webhook 地址**，长这样：
   ```
   https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```
6. **整条复制下来**（包括 `https://` 到结尾的 key 全部），先贴到电脑的记事本里待用

> ⚠️ **这条链接相当于"群机器人的钥匙"，千万别外发、别贴朋友圈、别提交到公开代码里。** 下一步我们会把它藏进 GitHub 的保险箱。

---

## 步骤 3️⃣：在 GitHub 仓库里存"保险箱密钥"（Secrets）

1. 打开你刚导入的仓库（比如 `https://github.com/你的用户名/my-trendradar`）
2. 顶部菜单点 **Settings**（设置）
3. 左侧菜单找 **Secrets and variables** → **Actions**
4. 点绿色按钮 **New repository secret**
5. 添加下面这条（**必填**）：

   | Name（变量名，必须一字不差） | Secret（值） |
   |------|------|
   | `WEWORK_WEBHOOK_URL` | 粘贴步骤 2️⃣ 复制的 webhook 地址 |

6. （可选）如果你想开启 **AI 分析功能**（让 AI 总结热点趋势），再加几条：

   | Name | Secret | 说明 |
   |------|------|------|
   | `AI_API_KEY` | 你的 AI API Key | 比如 DeepSeek、OpenAI、通义千问等 |
   | `AI_MODEL` | `deepseek/deepseek-chat` | 模型名，按服务商填 |
   | `AI_ANALYSIS_ENABLED` | `true` | 开启 AI 分析总开关 |

   > 💡 **不配 AI 也能正常用**，只是推送里不会有"AI 点评"，纯关键词匹配的热点。想先跑通再说的话，这几条可以先跳过。

7. 每条都点 **Add secret** 保存

**验证**：保存后，Secrets 列表里应该能看到 `WEWORK_WEBHOOK_URL` 这一行（值会显示 `***` 隐藏）。

---

## 步骤 4️⃣：开启 GitHub Actions 自动运行

1. 回到仓库首页，顶部菜单点 **Actions**（如果弹出黄色提示"Workflows are disabled"，点绿色按钮 **I understand my workflows, go ahead and enable them**）
2. 左侧会出现 3 个 workflow：
   - **Get Hot News**（主力：每天早 8 晚 8 自动抓热点 + 推送）
   - **Check In**（签到：每 7 天手动跑一次续期，见下方说明）
   - **issue-guard**（可忽略）
3. 点 **Get Hot News**，右上角 **Run workflow** → 下拉选 `main` 分支 → 点绿色 **Run workflow**
4. 等 1~2 分钟，刷新页面，如果看到绿色 ✅ 就成功了；红色 ❌ 去下方"常见问题"排查
5. 跑成功后，你的企业微信群应该收到一条热点推送！🎉

### ⚠️ 关于 7 天签到机制

这个项目的免费版 GitHub Actions 工作流内置了"7 天续期"机制（防止被滥用）。**每 7 天你需要手动跑一次 "Check In" workflow**，不然 "Get Hot News" 会自动停止。

**续期方法**：Actions → 左侧点 **Check In** → **Run workflow** → 绿色按钮，完事。

**建议**：在手机日历里设一个每周重复的提醒，"周一上午跑一次 Check In"。

---

## 步骤 5️⃣：开启 GitHub Pages 在线网页版

每次推送后，项目会把热点报告写成一个漂亮的 HTML 网页 (`index.html`) 推回仓库。开启 Pages 后，你会得到一个公开的在线地址（比如 `https://你的用户名.github.io/my-trendradar`），手机打开也能看。

1. 仓库里 **Settings** → 左侧 **Pages**
2. **Source**（源）选 **Deploy from a branch**
3. **Branch** 选 `main` / `(root)`，点 **Save**
4. 等 2~5 分钟，刷新页面，顶部会出现你的 Pages 地址
5. 点进去就能看到在线报告（首次访问可能要等首次 "Get Hot News" 跑完生成 index.html）

> 💡 如果你把仓库设成 **Private**（私有），GitHub Pages 默认要 GitHub Pro 会员才能开。想白嫖 Pages 的话，可以把仓库改成 Public（公开）——只要 secrets 都放在保险箱里，webhook 就不会泄露。

## 🔧 常见问题排查

### ❓ Actions 跑出红色 ❌，点进去看报错

**常见报错 1：`Config missing`**
→ 说明 `config/config.yaml` 没跟着推上去。检查仓库里这个文件是否存在。

**常见报错 2：`推送失败 / webhook invalid`**
→ `WEWORK_WEBHOOK_URL` 填错了，或者机器人被群主移除了。重新复制 webhook，在 Secrets 里点 **Update**。

**常见报错 3：`⚠️ 试用期已结束`**
→ 7 天续期到了，去 Actions → **Check In** → **Run workflow** 签到。

**常见报错 4：`ai_filter` 或 `ai_analysis` 相关报错**
→ 开了 AI 但 `AI_API_KEY` 没填或填错。要么去 Secrets 里填，要么把 `config/config.yaml` 里 `ai_analysis.enabled` 保持 `false`。

### ❓ 企业微信没收到推送，但 Actions 是绿色的

1. 检查群机器人没被管理员关闭
2. 去机器人"消息记录"里看是否有收到（有时候被折叠到"机器人消息"子菜单）
3. 检查关键词是否太窄——当前配置 5 组关键词都是广州小升初专题，如果当下热榜没有相关新闻，推送内容就是空的。可以编辑 `config/frequency_words.txt` 加几个更泛的词（比如"教育"）临时测试

### ❓ 想改推送时间

编辑仓库里的 `.github/workflows/crawler.yml`，找到这两行：
```yaml
- cron: "0 0 * * *"      # UTC 00:00 = 北京 08:00
- cron: "0 12 * * *"     # UTC 12:00 = 北京 20:00
```
改 cron 表达式即可。格式 `"分 时 日 月 星期"`，时间用 **UTC**（北京时间 -8 小时）。
> 比如想改成北京早 7 晚 9：UTC 23:00 和 UTC 13:00 → `"0 23 * * *"` 和 `"0 13 * * *"`

### ❓ 想加/减关键词

编辑仓库里 `config/frequency_words.txt`，参照文件开头的注释语法。改完 push 到 main 分支，下次自动运行就生效。

### ❓ 想加多个微信群

企业微信群机器人支持"多 webhook 用分号分隔"。在 Secrets 里把 `WEWORK_WEBHOOK_URL` 的值改成：
```
https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=第一个;https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=第二个
```

### ❓ 想关闭整个推送（不想删项目）

Actions → **Get Hot News** → 右上角 `···` → **Disable workflow**。想恢复再点 **Enable**。
